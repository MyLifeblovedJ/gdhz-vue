# AI 技能记忆清单（welcome-5173）

## 目标
- 固化本项目常用技能触发条件，减少每次重复沟通。
- 适用于海洋灾害决策大屏前端（Vue + Cesium）。

## 必用流程技能
1. `using-superpowers`
- 触发：每次新会话开始。
- 作用：先检查是否存在可用技能，再行动。

2. `brainstorming`
- 触发：涉及视觉风格、布局结构、交互形态等创意决策时。
- 作用：先收敛方案，再改代码。

3. `verification-before-completion`
- 触发：准备声明“已完成”前。
- 作用：必须用构建/运行结果证明改动可用。

## 界面与实现技能
4. `interface-design`
- 触发：大屏布局、配色系统、信息层级、组件视觉规范。
- 本项目重点：
  - 深海政务感（权威、克制、低认知负担）
  - 头部导航 + 右侧演示工具栏 + 全屏地图
  - 避免赛博风和高噪声特效

5. `frontend-developer`
- 触发：Vue 组件拆分、状态管理、样式落地。
- 本项目重点：
  - `MapStage / ShellHeader / RightTools`
  - 2D/3D 切换
  - 响应式布局与层级管理

6. `frontend-design`
- 触发：需要进一步打磨视觉质量时。
- 本项目重点：
  - 海洋色 Token 统一
  - 交互态（hover/active）克制表达

## 协作与外部讨论技能
7. `ask`
- 触发：需要把上下文转发给 Gemini 做并行设计讨论。
- 约束：仅发送“收敛目标 + 硬约束 + 输出结构”，避免泛化跑题。

8. `pend`
- 触发：查询 Gemini 最新回复并回收结果。
- 作用：同步外部建议，再与本地方案合并。

## 当前项目硬约束（执行时必须检查）
- 使用 Vue 开发。
- 业务数据禁止写死，全部来自 Mock/API。
- 必须维护接口文档，支持同事接手。
- 代码注释使用中文。

- 左侧区域：暂时去掉，后续再讨论。
- 右侧工具栏：仅演示态（样式优先，功能后置）。
- 底部横幅：暂不展示。

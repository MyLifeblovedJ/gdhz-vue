# 海洋灾害决策大屏：配色与页面架构基线

## 1. 风格定位
- 目标：权威、克制、重点突出、低认知负担。
- 场景：领导决策大屏，优先可读性和信息优先级，不采用赛博炫光风。

## 2. 配色原则
- 主色：深海蓝系（背景与结构层）
- 强调色：亮海蓝（交互与焦点）
- 蒙版：深色半透明（保障文字可读）
- 语义告警：蓝/黄/橙/红分级

## 3. 核心页面架构
- 顶部：系统标题 + 主导航（首页、风浪潮、赤潮、海啸、咸潮入侵、海水入侵、海岸侵蚀、海平面上升、溢油）
- 左侧：三块主数据面板，统一蒙版底
- 右侧：Windy 风格垂直工具条，首个菜单按钮做差异化样式
- 中央：Cesium 地图主舞台，铺满全局
- 底部：暂不展示横幅
- 交互：2D/3D 一键切换，默认 3D

## 4. 组件原则
- 卡片圆角：8px
- 动画时长：200-300ms，仅用于状态反馈
- 表格：优先展示关键列，减少视觉噪声
- 注释：中文

## 5. 数据原则
- 业务内容只从 Mock/API 获取
- 组件只负责展示，不内嵌业务常量

# 首页重点站点与海堤值守设计

> 日期：2026-03-06
> 范围：`apps/gdhz-vue-2999`

## 目标

修复首页两个问题：

- 设备与数据行记录中的告警、预警站点缺少明确数值说明，需要补充“超警/距警戒”的具体值。
- 首页底图缺少重点站点，需要使用 mock 数据与接口方式渲染站点，并实现表格、地图、站点详情、海堤风险卡联动。

## 设计结论

首页采用首页专用三段 dock：

- 左侧：当前选中站点详情
- 中间：重点站点数据表
- 右侧：海堤风险值守卡

保留首页现有左右列与地图总体结构，不重写主页面骨架。

## 数据模型

首页数据按四层关系组织：

- 站点：风暴潮核定站、潮位站、浪高站
- 岸段：逻辑分区，一个市可对应一到两个岸段
- 海堤：归属于岸段
- 断点：海堤内重点监测断点，展示堤顶高程、预测潮位、当前潮位、漫堤或距堤顶值

站点与岸段关联，岸段下挂海堤，海堤下挂断点。右侧风险卡展示优先级为“当前站点关联岸段 -> 重点海堤 -> 重点断点”。

## 交互设计

- 首页默认选中规则：
  - 优先第一个告警站点
  - 其次第一个预警站点
  - 最后第一个在线重点站
- 点击中间表格行：
  - 更新当前选中站点
  - 地图定位并高亮站点
  - 左侧详情卡更新
  - 右侧海堤风险卡切换到该站点关联岸段
- 点击地图站点：
  - 反向更新中间表格选中行
  - 更新左侧详情卡
  - 更新右侧海堤风险卡

## Mock 接口

- `homeApi.getHomeMonitoringData()`
  - 返回首页重点站点、岸段、海堤、断点全量数据
- 设备列表仍保留原有 `deviceApi.getDevices()` 逻辑

## 展示规则

设备与数据列表中的异常站点文案补充明确数值：

- 风暴潮核定站：`超警 18cm`
- 潮位站：`距警戒 6cm`
- 浪高站：`超预警 0.4m`

右侧海堤风险卡展示：

- 当前关联岸段摘要
- 重点海堤列表
- 重点断点列表

断点级说明优先使用：

- `漫堤 +0.12m`
- `距堤顶 -0.18m`

## 验收标准

- 首页地图可见重点站点
- 点击表格行可定位并高亮地图站点
- 点击地图站点可反向联动表格与详情
- 左侧设备与数据列表中的异常行出现具体数值说明
- 右侧值守卡展示海堤与断点风险，不再重复设备统计
- 全流程使用 mock 数据与接口方式，不直接硬编码在页面组件内

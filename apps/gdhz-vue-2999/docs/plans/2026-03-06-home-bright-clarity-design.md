# 2999 首页亮色清晰化设计

> 日期：2026-03-06
> 范围：`apps/gdhz-vue-2999`

## 目标

将当前“亮色底 + 深海科技残留”的混合风格，收敛为“明亮、中性、清晰、克制”的指挥界面。重点解决：
- 文字偏青灰、层级不清
- 标题和导航装饰过重
- 全局令牌与组件硬编码脱节
- 小字号过多导致信息发虚

## 设计判断

### 方向选择

采用“明亮中性指挥界面”，不再继续强化海洋青蓝科技风。

原因：
- 当前用户目标是“明朗、简洁清晰、一目了然”
- 大量青蓝文字和发光装饰会削弱可读性，强化主题噪音
- 指挥类产品要优先建立信息确定性，而不是风格存在感

### 新的气质关键词

- 明亮
- 中性
- 稳定
- 克制
- 高信息确定性

### 不再保留的旧语义

- 大面积青蓝文字
- 面板标题统一使用高饱和强调色
- 玻璃高光、液态高光、顶部发光线作为默认手段
- 10px/11px 作为广泛正文与标签字号
- 过多渐变字、发光字、装饰性波浪视觉

## 视觉系统调整

### 1. 文字体系

目标：文字首先是信息，不再承担海洋氛围渲染。

策略：
- `--text-primary` 改为中性深墨色，用于核心标题与主要信息
- `--text-secondary` 改为中性深灰，用于正文和高频说明
- `--text-tertiary` 改为中性灰，用于标签和次级信息
- `--text-muted` 仅用于时间戳、辅助说明、极弱提示
- 禁止正文默认带青蓝色相，蓝色只在链接、激活或特定高亮中出现

### 2. 强调色体系

目标：强调色只服务状态和焦点，不作为默认文字色。

策略：
- 面板默认标题改为主文字色，而不是 `accent-cyan`
- 保留青色系作为交互焦点和少量关键按钮/选中态
- 红橙黄蓝预警色保留，用于业务状态
- 降低 glow、透明描边和渐变高光的使用密度

### 3. 字号体系

目标：提升清晰度和阅读确定性。

策略：
- 将 10px/11px 的使用范围压缩到极少数角标
- 标签/辅助正文尽量提升到 12px
- 核心正文稳定在 13px-14px
- 面板标题避免过细过小，减少 `uppercase` 依赖

### 4. 面板与导航

目标：结构清晰，视觉不抢。

策略：
- 面板顶部发光线改为更弱或按需出现
- 面板标题默认用中性文字，状态通过角标/边框/局部标签表达
- 导航去掉液态玻璃主导感，改为更克制的选中态
- Header 品牌区从“装饰图形驱动”改为“文字与秩序驱动”

- 首页主文字不再普遍偏青灰
- 顶部导航和品牌区显著更清晰、更克制
- 面板标题不再大面积使用青色强调
- `TyphoonInfo` 中的文字层级和标签可读性明显提升
- 全局文档与当前视觉令牌一致，不再保留旧深海系统定义

# 2999 导航与图例精修设计

> 日期：2026-03-06
> 范围：`apps/gdhz-vue-2999`

## 目标

在现有亮色清晰化基础上继续解决两个问题：
- 顶部导航过于小气，仍带组件感，不够高级
- 图例位置像地图插件，缺乏与左侧业务面板的关系

## 设计结论

### 1. 导航改为 Briefing Rail

不再使用任何玻璃胶囊或浮动容器。

新的导航是“简报章节栏”：
- 容器本身透明，不承担块状存在感
- 通过一条极细基线组织导航项
- 激活态用更稳的字重、深色文本和短下划线表达
- 普通态保持清晰，但不靠蓝灰透明度制造层级
- 适当增加横向呼吸感，让顶部更像产品秩序，而不是组件集合

### 2. 图例改为 Left Margin Note

图例不再沉在左下角，而是成为左列设备区的“侧注”。

策略：
- 保持浮层能力，但视觉上靠近 `设备与数据`
- 位置放在左侧大列右边缘之外的地图留白区
- 垂直位置对齐到设备区下半段，而不是底部 dock 保留区
- 样式与当前亮色系统一致，尺寸略收紧，像注释卡而不是工具盒

### 3. 头部继续减重

Header 本身不再强化面板感：
- 降低白玻璃厚度
- 让导航成为头部主角
- 品牌、时间、天气作为稳定辅助信息存在

## 替代的默认项

- 默认玻璃 segmented nav -> briefing rail
- 默认地图角落 legend -> device-note legend
- 默认容器强调 -> 信息关系强调

## 验收标准

- 导航不再像悬浮玻璃组件
- 导航横向节奏更大，激活态更稳更高级
- 图例出现在 `设备与数据` 的右侧地图留白中
- 图例不遮挡两侧主面板和工具栏
- 构建通过

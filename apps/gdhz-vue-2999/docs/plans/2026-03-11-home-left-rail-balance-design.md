# Home Left Rail Balance Design

> 日期：2026-03-11
> 范围：`apps/gdhz-vue-2999`

## 目标

解决首页左列空间分配不均的问题：

- `当前预警` 在大屏上不再用固定较高的可视高度，而是按首页视口分档控制默认可见条数
- `海堤风险` 获得更高的空间优先级，保证大屏下也能稳定看到 2 到 3 条完整记录
- `设备与数据` 同步按分档控制默认可视容量，超出部分继续滚动
- `智能决策` 移除 `态势研判 / 应急响应 / 数据巡检` 推荐按钮，收紧面板高度

## 设计结论

保持首页两列与地图结构不变，只调整首页布局参数和局部面板内容：

- 通过 `homeViewportProfile` 为 `spacious / standard / compact` 三档增加首页左列与右列的展示参数
- `HomeOverview.vue` 根据当前窗口高度读取这些参数，并通过 CSS 变量驱动 `当前预警`、`设备与数据`、`海堤风险` 的高度策略
- `AIDecisionPanel.vue` 删除建议按钮区，只保留摘要与输入框

## 展示规则

- `当前预警`
  - 大屏：默认 4 条
  - 中屏：默认 3 条
  - 紧凑屏：默认 2 条
  - 超出后内部滚动
- `设备与数据`
  - 大屏：默认显示更多设备行
  - 中屏：适中
  - 紧凑屏：较少
  - 超出后内部滚动
- `海堤风险`
  - 保持弹性填充
  - 新增最小可视高度，避免被 `当前预警` 和 `智能决策` 挤压到只剩半条

## 验收标准

- 27 寸屏幕下，`海堤风险` 默认可见区域至少能完整看到 2 到 3 条记录
- `当前预警` 在不同屏幕高度下可视条数变化明显，超出部分滚动
- `设备与数据` 在不同屏幕高度下默认可视容量随之变化
- `智能决策` 不再出现 `态势研判 / 应急响应 / 数据巡检` 三个按钮

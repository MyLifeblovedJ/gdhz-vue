# 首页全量设备与台风 2D/3D 一致显示设计

**背景**

`2999` 首页当前存在两个结构性问题：

1. 全量设备只在 Leaflet 2D 中渲染，Cesium 3D 没有对应实体，所以切到 3D 后会丢失设备。
2. 台风真实数据已经接入，但渲染逻辑仍主要停留在 2D 图层；3D 下没有台风路径、风圈和台风标记，导致首页默认 3D 时看不到台风。

用户要求：

- 首页默认保持 `3D`
- 无论 `2D` 还是 `3D`，都必须看到全量设备和真实台风
- 去掉首页“重点站点”在地图层的专用渲染
- 风圈不能退回圆形，需按真实四象限半径绘制

## 方案选择

### 方案 A：共享视觉模型 + 双端渲染器

抽出一层纯函数视觉模型：

- 设备：输出颜色、尺寸、标签、坐标
- 台风：输出历史路径、预报路径、风圈多边形顶点、当前中心点

Leaflet 和 Cesium 都消费同一份结果，各自负责落到具体图层/实体。

优点：

- 2D/3D 表达一致
- 风圈几何只实现一次
- 适合补测试，回归成本低

缺点：

- 需要改动 `MapContainer.vue`

### 方案 B：只给 3D 补独立逻辑

直接在 `MapContainer.vue` 里新增一套 Cesium 设备和台风渲染，不抽共享几何。

优点：

- 初始开发快

缺点：

- 2D/3D 会继续分叉
- 风圈与路径样式容易再次不一致

## 结论

采用方案 A。

## 设计要点

### 1. 台风视觉模型

新增纯函数工具，负责：

- 返回台风强度颜色
- 把 `radius7/10/12` 四象限半径转成闭合多边形顶点
- 输出历史路径线段、预报路径点列和当前中心点

这样 Leaflet 用 `L.polyline/L.polygon`，Cesium 用 `polyline/polygon/billboard or point`。

### 2. 设备视觉模型

新增纯函数工具，负责：

- 根据设备状态输出颜色与尺寸
- 归一化 2D/3D 共用的视觉字段

Leaflet 保留现有 marker 风格；Cesium 新增全量实体渲染。

### 3. MapContainer 重构

保留现有 2D 设备渲染，但补足 3D：

- 新增全量设备实体数组与清理逻辑
- 新增台风实体数组与清理逻辑
- 默认进入 3D 时，直接渲染 Cesium 设备和台风
- 切换 2D/3D 时两侧都保留自己的内容，不依赖另一个模式

### 4. 首页地图交互

移除地图上的 `homeStations` 专用图层与点击事件，不再把首页重点站点当成地图主数据。

首页主地图统一使用：

- store 全量 `devices`
- store 真实 `typhoonData`

### 5. 验证策略

先写失败测试，覆盖：

- 首页默认地图模式仍为 `3D`
- 台风风圈几何生成的是四象限多边形，不是圆
- 台风视觉模型能产出 3D/2D 都可消费的风圈和路径数据
- 设备视觉模型能给出稳定的颜色与尺寸

然后再补渲染实现。

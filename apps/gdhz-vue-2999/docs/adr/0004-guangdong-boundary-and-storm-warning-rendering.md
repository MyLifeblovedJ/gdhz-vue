# ADR-0004: 广东省界与风暴潮分级边界渲染规范

## Status

Accepted

## Date

2026-03-04

## Context

本项目地图需要同时满足三项要求：

1. 省级边界必须使用独立数据源，并避免与市级边界混线。
2. 风暴潮预警必须按城市级别映射到边界范围（红/橙/黄/蓝）。
3. 在深色和浅色底图下均保持可读性。

为此，统一采用“省界置顶 + 城市分级填充 + 城市单色细边界”的渲染策略。

## Decision

### 1) 数据源

- 省界：`src/data/广东省line.json`
- 市界：`src/data/GDSJXZQH.json`
- 预警等级来源：`store.alerts` 中 `type='surge'` 的城市级预警数据

### 2) 渲染规则

- 省界：亮黄色 `#ffea00`，2D/3D 线宽 `2.5`，且渲染层级高于市界。
- 受灾城市填充色：
  - red: `#FF4D4F`
  - orange: `#FF9C6E`
  - yellow: `#FADB14`
  - blue: `#1677FF`
- 填充透明度：`0.55`
- 受灾城市边界：单一灰色 `#777777`，2D `0.8`，3D `1.2`。
- 不绘制“无预警城市”边界。

### 3) 层级策略（避免省界与市界混线）

- Leaflet：
  - `city-warning-pane`（低层）
  - `province-boundary-pane`（高层）
- Cesium：先渲染城市面/线，再渲染省界线，确保省界视觉置顶。

## Non-Negotiable Guardrails

后续改动禁止：

1. 将省界回退为从市界推导外轮廓。
2. 重新绘制无预警城市边界干扰预警表达。
3. 将省界层级降到市界之下导致混线。

## Verification Checklist

1. 广东省界仅使用亮黄色，且连续完整。
2. 仅有预警城市显示填充及边界。
3. 红橙黄蓝四级可清晰区分。
4. 浅色与深色底图下均可识别。
5. `npm run build` 通过。

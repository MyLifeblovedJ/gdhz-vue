# ADR-0006: 区县边界按需加载与沿海过滤显示

## Status

Accepted

## Date

2026-03-04

## Context

新增区县边界数据 `src/data/GDQXXZQH.json` 后，存在三个问题：

1. 缩放放大时区县数据加载慢，影响交互体验。
2. 市级与区县名称出现重复（例如“中山/中山市”）。
3. 非沿海城市区县不应显示（业务聚焦海洋灾害）。

## Decision

在 `MapContainer.vue` 采用以下策略：

1. 区县 GeoJSON 改为按需懒加载（dynamic import），不进入首屏主包。
2. 仅当达到显示阈值时触发加载与渲染：
   - 2D：边界 `zoom >= 9`，名称 `zoom >= 10`
   - 3D：边界 `camera height <= 800000`，名称 `<= 500000`
3. 对区县按“沿海城市范围”过滤：
   - 使用区县质心与沿海地市范围进行空间判断。
   - 非沿海区县不渲染。
4. 标签去重：
   - 区县名称做规范化（去掉“市”后缀）；
   - 若与沿海市级显示名重复（如“中山”），区县标签不显示。
5. 渲染缓存：
   - 对 2D/3D 区县图层与标签建立已渲染标记，避免重复构建实体。

## Consequences

1. 首屏加载压力降低，放大时区县显示响应更稳定。
2. 标签层级语义更清晰，不再出现“中山/中山市”重复。
3. 地图信息更聚焦沿海业务范围。

## Verification Checklist

1. 刷新后未放大前不加载区县数据 chunk。
2. 到达阈值后显示区县边界和名称（若有）。
3. 不显示非沿海区县。
4. 不再出现“中山/中山市”重复标签。
5. `npm run build` 通过。
